
# Diffusion Policy for Dual-Arm Manipulation

This repository contains an implementation of **Diffusion Policy** for dual‑arm manipulation tasks in [robosuite](https://robosuite.ai/).  
The policy generates action chunks via iterative denoising using a conditional U‑Net, trained on scripted demonstrations.

## Repository Structure

```

├── diffusion_policy/               # Core diffusion policy modules
│   ├── diffusion.py                # DDPM scheduler (q_sample, sampling loop)

│   └── model.py                    # Conditional U‑Net 1D architecture
├── preprocessing/                  # Data preparation scripts
```

> The `diffusion_policy/` directory contains `diffusion.py`, `evaluate_policy.py`, and `model.py`.

## Data Preparation

Your raw demonstrations should be stored as an HDF5 file generated by robosuite (e.g., `script_collect_demo2.hdf5`).  
The preprocessing step:

1. Loads all demos from the HDF5 file.
2. Splits demos into train/val (80/20 by default).
3. Creates sliding windows of observations and action chunks.
4. Normalizes observations and actions using training set statistics.
5. Saves the processed data and normalisation stats.

Run:

```bash
cd preprocessing
python preprocess_data.py \
    --raw_path ../data/script_collect_demo2.hdf5 \
    --output_dir ../data/processed_scripted \
    --obs_horizon 2 \
    --action_chunk 2 \
    --train_ratio 0.8
```

This will create:
- `diffusion_h2_c2_data.npz` – training/validation windows
- `norm_stats.npz` – mean and std for obs and actions
- `preprocess_summary.txt` – metadata

## Training

Train the diffusion model using the prepared dataset:

```bash
cd preprocessing   # train_diffusion.py is in preprocessing/
python train_diffusion.py \
    --data_path ../data/processed_scripted/diffusion_h2_c2_data.npz \
    --norm_stats_path ../data/processed_scripted/norm_stats.npz \
    --output_dir ./checkpoints_c2 \
    --epochs 200 \
    --batch_size 64 \
    --num_diffusion_steps 100 \
    --dropout 0.1 \
    --data_augment \
    --noise_std 0.005
```

Key arguments:
- `--data_augment` adds Gaussian noise to observations and actions.
- `--dropout` helps against overfitting (default 0.1).
- `--early_stop_patience` stops training if validation loss does not improve.

The training script saves:
- `best_diffusion_model.pth` – best checkpoint based on validation loss.
- `training_history.json` – train/val losses per epoch.
- `loss_curve.png` – plot of training and validation loss curves.

## Evaluation

Evaluate a trained policy in the robosuite environment:

```bash
cd diffusion_policy
python evaluate_policy.py \
    --model_path ../preprocessing/checkpoints_c2/best_diffusion_model.pth \
    --source_hdf5_path ../data/script_collect_demo2.hdf5 \
    --num_episodes 10 \
    --max_steps 600 \
    --save_screenshots
```

The script will:
- Load the model and normalisation statistics (saved in the checkpoint).
- Run `num_episodes` episodes in the `TwoArmLift` environment.
- Output success/failure per episode.
- Generate evaluation plots (`success_rate.png`, `steps_histogram.png`, `steps_per_episode.png`) and a JSON summary.
- Optionally save screenshots of successful episodes (max 5).

## Key Files Description

| File | Purpose |
|------|---------|
| `diffusion.py` | DDPM scheduler: forward diffusion (`q_sample`) and reverse sampling loop. |
| `model.py` | Conditional U‑Net 1D with sinusoidal time embedding, residual blocks, and FiLM‑like conditioning. |
| `train_diffusion.py` | Training loop with MSE noise prediction, data augmentation, early stopping, and loss plotting. |
| `evaluate_policy.py` | Closed‑loop evaluation, state extraction, success detection, and result visualisation. |
| `preprocess_data.py` | Converts raw HDF5 demos into sliding‑window datasets for diffusion training. |
| `utils.py` | Helper functions: load robosuite HDF5, normalise observations/actions. |

## Notes

- The policy uses **full MuJoCo state** (50‑dim) as observation, not images.
- Action space: 14‑dim delta joint commands for two Panda arms.
- The environment is expected to provide a `_check_success()` method (robosuite standard).
- Normalisation statistics are computed per action dimension (not global scaling).

## Results

With action chunk length = 2, dropout = 0.1, data augmentation, and 30 demonstrations, the policy achieved **50% success rate** over 10 episodes (5 successes).  
Failed episodes run to the timeout (600 steps). Longer action chunks (4 or 8) led to 0% success due to overfitting.
